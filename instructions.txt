Content Update Instructions
===========================

This app stores content in 2 places:

1. In MySQL
- Main homepage text and contact info are in `site_content`
- Room cards are in `rooms`
- Gallery cards are in `gallery_items`

2. In files inside the repo
- Some labels and static text are hardcoded in `web/templates/index.html`
- Global styling and background rules are in `web/static/styles.css`
- All public images should be uploaded directly into `web/static/uploads/`


1. Backup before changing content
---------------------------------

If you run the app with Docker Compose, create a database backup first:

```bash
docker compose exec -T db sh -lc 'mysqldump -uroot -p"$MYSQL_ROOT_PASSWORD" "$MYSQL_DATABASE"' > backup_$(date +%F_%H%M%S).sql
```


2. How to change text stored in the database
-------------------------------------------

The easiest way is to run MySQL commands inside the Docker database container.

Check current homepage content:

```bash
docker compose exec -T db sh -lc 'mysql -uroot -p"$MYSQL_ROOT_PASSWORD" "$MYSQL_DATABASE" -e "
SELECT id, brand_name, tagline, hero_title, hero_subtitle, season_note, booking_url, address, phone, email, check_in, check_out, about_title, about_body
FROM site_content;
"'
```

Check current room cards:

```bash
docker compose exec -T db sh -lc 'mysql -uroot -p"$MYSQL_ROOT_PASSWORD" "$MYSQL_DATABASE" -e "
SELECT id, name, summary, capacity, size_sqm, room_count, feature_note, image_path
FROM rooms
ORDER BY id;
"'
```

Check current gallery cards:

```bash
docker compose exec -T db sh -lc 'mysql -uroot -p"$MYSQL_ROOT_PASSWORD" "$MYSQL_DATABASE" -e "
SELECT id, title, subtitle, tone, image_path
FROM gallery_items
ORDER BY id;
"'
```

Example: update the main homepage text:

```bash
docker compose exec -T db sh -lc 'mysql -uroot -p"$MYSQL_ROOT_PASSWORD" "$MYSQL_DATABASE" -e "
UPDATE site_content
SET
  brand_name = \"Your Hotel Name\",
  tagline = \"Short tagline\",
  hero_title = \"Main title on the first screen\",
  hero_subtitle = \"Short text under the main title\",
  season_note = \"Season note\",
  booking_url = \"https://your-booking-link.example\",
  address = \"Your address\",
  phone = \"+7 ...\",
  email = \"hello@example.com\",
  check_in = \"14:00\",
  check_out = \"12:00\",
  about_title = \"About section title\",
  about_body = \"About section text\"
WHERE id = 1;
"'
```

Example: update one room:

```bash
docker compose exec -T db sh -lc 'mysql -uroot -p"$MYSQL_ROOT_PASSWORD" "$MYSQL_DATABASE" -e "
UPDATE rooms
SET
  name = \"Family Room\",
  summary = \"Short room description\",
  capacity = 4,
  size_sqm = 20,
  room_count = 1,
  feature_note = \"Good for families\"
WHERE id = 1;
"'
```

Example: update one gallery item:

```bash
docker compose exec -T db sh -lc 'mysql -uroot -p"$MYSQL_ROOT_PASSWORD" "$MYSQL_DATABASE" -e "
UPDATE gallery_items
SET
  title = \"Pool\",
  subtitle = \"Short gallery description\",
  tone = \"aqua\"
WHERE id = 1;
"'
```


3. Where to upload images
-------------------------

Upload all images directly into:

```text
web/static/uploads/
```

Suggested structure:

```text
web/static/uploads/
  hero/
  rooms/
  gallery/
  backgrounds/
```

Examples:

```text
web/static/uploads/hero/main.jpg
web/static/uploads/rooms/room-1.jpg
web/static/uploads/gallery/pool.jpg
web/static/uploads/backgrounds/site-bg.jpg
```

Public URLs for these files will be:

```text
/static/uploads/hero/main.jpg
/static/uploads/rooms/room-1.jpg
/static/uploads/gallery/pool.jpg
/static/uploads/backgrounds/site-bg.jpg
```

Recommended file naming:
- Use ASCII filenames
- Avoid spaces
- Prefer names like `hero-main.jpg`, `room-1.jpg`, `pool.jpg`
- Prefer `.jpg`, `.jpeg`, `.png`, or `.webp`


4. How to connect uploaded images to the site
---------------------------------------------

Uploading a file is not enough. You must also save its public URL in the database.

Example: set the main hero image:

```bash
docker compose exec -T db sh -lc 'mysql -uroot -p"$MYSQL_ROOT_PASSWORD" "$MYSQL_DATABASE" -e "
UPDATE site_content
SET hero_image_path = \"/static/uploads/hero/main.jpg\"
WHERE id = 1;
"'
```

Example: set the image for room `id = 1`:

```bash
docker compose exec -T db sh -lc 'mysql -uroot -p"$MYSQL_ROOT_PASSWORD" "$MYSQL_DATABASE" -e "
UPDATE rooms
SET image_path = \"/static/uploads/rooms/room-1.jpg\"
WHERE id = 1;
"'
```

Example: set the image for gallery item `id = 1`:

```bash
docker compose exec -T db sh -lc 'mysql -uroot -p"$MYSQL_ROOT_PASSWORD" "$MYSQL_DATABASE" -e "
UPDATE gallery_items
SET image_path = \"/static/uploads/gallery/pool.jpg\"
WHERE id = 1;
"'
```


5. How to change the main hero image
------------------------------------

The large image on the first screen is controlled by `site_content.hero_image_path`.

Set a hero image:

```bash
docker compose exec -T db sh -lc 'mysql -uroot -p"$MYSQL_ROOT_PASSWORD" "$MYSQL_DATABASE" -e "
UPDATE site_content
SET hero_image_path = \"/static/uploads/hero/main.jpg\"
WHERE id = 1;
"'
```

Remove the hero image and go back to the built-in gradient artwork:

```bash
docker compose exec -T db sh -lc 'mysql -uroot -p"$MYSQL_ROOT_PASSWORD" "$MYSQL_DATABASE" -e "
UPDATE site_content
SET hero_image_path = \"\"
WHERE id = 1;
"'
```


6. How to change the page background image
------------------------------------------

There are 2 different image areas:

1. Hero image
- Controlled by `site_content.hero_image_path`
- Best for the large image in the first screen

2. Whole page background
- Controlled by CSS in `web/static/styles.css`
- Right now the page uses gradients, not a background file

To set a full-page background image:

1. Upload an image to:

```text
web/static/uploads/backgrounds/site-bg.jpg
```

2. Edit the `body { ... }` block in `web/static/styles.css`

Replace the current `background:` with something like:

```css
background:
	linear-gradient(rgba(252, 247, 236, 0.72), rgba(247, 241, 230, 0.82)),
	url("/static/uploads/backgrounds/site-bg.jpg") center / cover fixed no-repeat;
```

If you only want to change the main visual in the first screen, use `hero_image_path` instead of changing the whole page background.


7. Text that is not stored in the database
------------------------------------------

Some text is hardcoded in:

```text
web/templates/index.html
```

Examples:
- Navigation labels
- Button labels
- Section labels like `О концепции`, `Номера`, `Галерея`, `Контакты`
- Amenity titles and descriptions
- Some helper text in the contact and snapshot blocks

If you want to change this text, edit `web/templates/index.html`.


8. When changes appear
----------------------

Database text changes:
- Usually appear after browser refresh

New uploaded images in `web/static/uploads/`:
- Usually appear after browser refresh

Template changes in `web/templates/index.html`:
- Restart the app

CSS changes in `web/static/styles.css`:
- Local run: browser hard refresh is often enough
- Docker Compose: rebuild/restart is safest

Docker Compose rebuild:

```bash
docker compose up --build -d
```

Local run:

```bash
go run ./cmd/server
```


9. Recommended update workflow
------------------------------

1. Upload images into `web/static/uploads/...`
2. Refresh the browser and verify the file path
3. Update database text and image paths
4. If needed, edit `web/templates/index.html` for hardcoded text
5. If needed, edit `web/static/styles.css` for colors or background image
6. Refresh the page
7. Restart or rebuild only if file changes do not appear


10. Most important places
-------------------------

`site_content`
- Main homepage text, contacts, and hero image path

`rooms`
- Room cards and room image paths

`gallery_items`
- Gallery cards and gallery image paths

`web/static/uploads/`
- Public image folder

`web/templates/index.html`
- Hardcoded labels and static text

`web/static/styles.css`
- Styling, colors, gradients, and full-page background
